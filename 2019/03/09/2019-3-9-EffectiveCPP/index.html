Effective C++ 总结备忘

2 尽量以const, enum, inline替换#define

  • 对于单纯常量，最好以const对象或emus替换#define
  • 对于形式函数的宏，最好改用inline函数替换#define

3 尽可能使用const

关键词const出现在左边表示被指物为const；关键词const出现在\右边表示指针本身是const

const 成员函数

  • const实施成员函数的目的，是为了确认该成员函可作用于const对象上。
  • 两个成员函数只是const属性不一样，也可以被重载。
  • const成员函数不可以改变对象内任何non-static成员变量。
  • 当const　和　non-const成员函数有着实质等价的实现时，可以用non-const调用const，避免代码重复。不能用const成员函数调用non-const成员函数，因为对象可能因此而改动。

４ 确定对象使用前已经被初始化

对内置对象需要进行手工初始化，因为c++不保证初始化他们。

对对象成员的初始化操作发生在进入构造函数本体之前。在构造函数的函数体内进行赋值操作并不能算是初始化，初始化开始的时间更早，发生于这些成员的default构造函数被调用之时。构造函数的最佳写法是用成员初始化列表来对对像的成员进行初始化操作。

为避免需要记住成员变量何时必须在成员初始化列表中初始化，何时不需要，最简单的做法是：总是使用成员初始化列表。

在初始化列表中，无论变量的顺序如何排列，class的成员变量总是以其声明的顺序来初始化，但为了表达清晰，还是以类中声明的顺序为好。基类成员的初始化总是先于派生类成员。

构造、析构、赋值运算符

几乎每一个类中都会有若干个构造函数、一个析构函数、一个赋值操作符(=)。确保这几个函数的正确性和稳定性对于程序的运行至关重要。

5 了解c++默认编写并调用哪些函数

当你创建一个空的对象时，编译器会默认为你的对象创建四个函数:

  • default 构造函数
  • copy　构造函数
  • copy assign　操作符
  • 析构函数

copy构造函数和copy assign操作符的编译器版本只是单纯的将源对象的每一个non-static成员变量拷贝到目标对象。另外，只要在对象的non-static成员允许赋值操作时，以及赋值操作有实际意义，编译器才会创建默认的copy　assign操作符。例如，在一个内含reference或const成员的类中定义赋值运算符，编译器认为是不允许的，便不会执行这个命令。
