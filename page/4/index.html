Effective C++ 总结备忘

2 尽量以const, enum, inline替换#define

  • 对于单纯常量，最好以const对象或emus替换#define
  • 对于形式函数的宏，最好改用inline函数替换#define

3 尽可能使用const

关键词const出现在左边表示被指物为const；关键词const出现在\右边表示指针本身是const

const 成员函数

  • const实施成员函数的目的，是为了确认该成员函可作用于const对象上。
  • 两个成员函数只是const属性不一样，也可以被重载。
  • const成员函数不可以改变对象内任何non-static成员变量。
  • 当const　和　non-const成员函数有着实质等价的实现时，可以用non-const调用const，避免代码重复。不能用const成员函数调用non-const成员函数，因为对象可能因此而改动。

４ 确定对象使用前已经被初始化

对内置对象需要进行手工初始化，因为c++不保证初始化他们。

对对象成员的初始化操作发生在进入构造函数本体之前。在构造函数的函数体内进行赋值操作并不能算是初始化，初始化开始的时间更早，发生于这些成员的default构造函数被调用之时。构造函数的最佳写法是用成员初始化列表来对对像的成员进行初始化操作。

为避免需要记住成员变量何时必须在成员初始化列表中初始化，何时不需要，最简单的做法是：总是使用成员初始化列表。

在初始化列表中，无论变量的顺序如何排列，class的成员变量总是以其声明的顺序来初始化，但为了表达清晰，还是以类中声明的顺序为好。基类成员的初始化总是先于派生类成员。

构造、析构、赋值运算符

几乎每一个类中都会有若干个构造函数、一个析构函数、一个赋值操作符(=)。确保这几个函数的正确性和稳定性对于程序的运行至关重要。

5 了解c++默认编写并调用哪些函数

当你创建一个空的对象时，编译器会默认为你的对象创建四个函数:

  • default 构造函数
  • copy　构造函数
  • copy assign　操作符
  • 析构函数

    强化学习解决TSP路径规划问题

    Posted on 2019-01-20 | Edited on 2019-09-24 | In 强化学习

    论文学习：强化学习解决TSP路径规划问题

    问题背景

    接触组合优化问题源于参加过的比赛的一些经历，例如华为软挑的云服务器资源分配问题，阿里巴巴调度大赛的虚拟机放置，都属于组合优化问题。传统解决组合优化问题的方法关键在于设计启发式函数，这种方法非常依赖于具体的问题背景，例如装箱/背包问题与旅行商邮路问题都是np-hard，但是面对的背景不一样。而且启发函数的好坏对结果影响很大，因而难以泛华。在大数据时代，可以考虑利用纯数据驱动的方法，利用深度学习、强化学习的理论来学习潜在的启发式方法，进而可以解决这一类问题而不是某一个问题。

    　算法模型

    我目前先针对比较容易理解的tsp问题来探索，TSP的定义很简单，一个二维平面上存在任意多个点，从给定的点出发，遍历所有点，最后回到起点，总的欧式距离最短，假设任意点间的权重仅与距离相关。回顾一下NLP中用来做机器翻译的Seq2Seq模型，输入是一个序列，输出是另外一个序列。这个思想可以很好的用在解决tsp上面，输入是任意的一组点集的排列，输出是得到的距离最短的排列，同样是序列到序列，而且两个序列是等长的。

    16年以来，业界利用强化学习＋深度学习的研究热点集中于与机器翻译类似的encode-decode框架，这种想法的开创者来自Google　DeepMind的论文。到了2017年，Google又提出了一种　Attention is all you need的结构，直接摈弃原来的双RNN模式，全部用attention取代，在机器翻译的效果甚至比原先的encode-decode还要好。

    因此，算法的主题框架还是借鉴了actor-critic模式。actor部分由注意力模型(attention model)执行——根据输入，选择一种排列方式并计算代价；critic是一种self-critic，由贪心选择得到的baseline函数来代表。训练方法是经典的蒙特卡洛采样－REINFORCEA算法。下面分别讲述各个部分。

    Attention-PG model

    定义一个包含n个顶点的实例集合s，每个点的特征由$x_i$表示，包含改点在二维平面中的坐标值。平面中的n个点构成了一个图，这个图是全连接的，包含自连接部分。定义一种遍历方法$\pi＝(\pi_1,…,\pi_n)$是所有点的一种全排列，对于任意$\pi_t\in{\pi_1,…,\pi_n}$以及$t\neq t’$,有$\pi_t \neq \pi_{t’}$. TSP的目标位对于给定的点集s给出一个遍历顺序，使总的距离最少。定义一个随机策略$p(\pi,s)$来根据给定的点集s选择遍历方法。将随机策略用参数$\theta$参数化再分解得到：

    编码器encoder生成所有输入点的embeding值。解码器decoder顺序的生成一个遍历序列$\pi$，每次输出一个点。decoder的输入为encoder的embeding值。对于TSP问题而言，当已经构建了部分遍历路径时，前面已经生成的点是不能再改变的，剩下的问题就是从已生成的最后一个点开始，在剩下还没有遍历的点中找到余下的路径然后回到起点。和已经遍历的点的顺序和坐标没有关系，只要知道了当前的最后一个点和第一个点就可以。decoder将第一个和最后一个点的embeding作为contexts参与下一个节点的解码过程，同时也会接受每个节点的mask标志，表示这个节点是否已经访问过。

    策略逼近和他的优势所在

    在策略梯度方法中，策略可以以任何方式来参数化，只要策略函数对参数$\theta$是可微分的。在本章节中，介绍在离散动作空间中最常用的参数化策略的方法并指针和动作值函数估计方法的相比，策略梯度的优势。策略梯度的方法同样也适用与连续动作空间。

    当动作是离散而且不大的时候，一个很自然的用来参数化的想法是数值性能来代表每一个动作－状态对的概率。在每个状态中有更高性能的动作被赋予更大的概率。比如可以用softmax函数来衡量动作的性能。

    使用参数化策略的方法的优势在于：

    • 可以输出确定的动作。相反在动作值估计的方法，使用$\epsilon－greedy$方法有一定的概率选择随机的动作。
    • 相比动作值函数，策略的函数逼近更容易
    • 可以将先验知识注入强化学习中
    • 策略梯度可以处理连续的动作空间，Ｑ-learning只能处理有限状态空间
    • 策略梯度直接对策略进行改进，基于值函数的方式都是根据动作状态值选择对应的动作输出，隐式的改善策略，而策略梯度的方法是一种直接改变策略的方法，更直接明了。

    通常来说，policy-gradient方法在Atari游戏中的变现要好于DQN模型。

    策略梯度的理论

    简单的梯度上升

    除了实际的相对与$\epsilon－greedy$动作选择的优势，策略梯度也有理论上的优势。由于连续的策略，作为函数要学习的参数，参数化的动作概率可以平滑的改变。更大的优势在于，相对于动作值的方法，策略梯度可以更大程度上保证收敛。

    本文想要以一种更直观的角度来讲解PG这种方法，尽量少的公式推导。

    • t 代表迭代次数，$\theta_t$ 到　$\theta_{t+1}$的过程称为两次迭代之间的更新

    Advantage Function and Baseline

    现在来讲一讲policy－gradient中另一个很重要的部分，优势函数A(s,a)和baseline。熟悉Q-learning的都知道，Q(s,a)代表在状态s追随策略$\pi_{\theta}$执行动作a后得到的累计折扣奖赏(cumulated discounted reward),　V(s)代表状态值函数，表示从状态s开始，跟随策略$\pi(s)$得到的累计折扣奖赏。

    如果仅仅用上述两者方法之一作为策略梯度的权重来更新参数，会出先一个问题：无法去除估计值的噪声对更新策略参数的影响。比如，有很多动作值比较接近，有100,101,102等等，在应用到更新梯度参数时，无法知道哪个动作是更好的，哪个是有问题的，因为在某一次中可能选中了100对应的动作而忽略了其他动作对应的值。

    在经典的REINFORCE算法中，将估计值Q(s,a)减去任意一个baseline函数都可以达到很好的效果，只要这个baseline与选取的动作无关，是独立的，甚至是随机变量都可以。非常推荐在policy-gradient中使用baseline方法，可以有效的减小梯度更新时的方差。

    神经网络解决组合优化问题

    这篇论文提出了一个利用神经网络结合强化学习的方法，解决组合优化这类np-hard问题。这篇文章将提出的方法应用在二类组合优化问题：TSP, knapsack, 都取得了不错的效果。这篇文章主要集中于研究TSP问题。训练一个循环神经网络，给定一个包含城市坐标点的数据集，对于包含所有不同城市的排列，预测一个概率分布，路程短的分步概率高。利用旅行长度的负数，作为奖励信号，利用model-free的policy-gradient来优化神经网络的参数。这篇文章比较了在训练集和测试集学习网络参数的不同。这篇文章的结果表明，尽管这种方法增加了计算开销，但没有工程上的任何启发式算法的设计，神经网络优化可以在有100个点的二维欧式图中得到近似最优结果。对于背包问题和其他
    NP-hard问题，同样的算法对最多达到200个物体的背包问题也得到了最优解。

    引言

    组合优化问题是计算机届的一个基础问题。一个非常的注明的例子就是旅行商问题。给定一个图，遍历图中的每一个点一次，最终回到起点，并使总的路程最短。如果遍历每一种可能的访问组合方式，所有的组合方式将是指数级的数量，无法在多项式的时间范围内求解。一般情况下，解决这种问题一般都是采用启发式的算法，类似遗传算法，蚁群之类。

    寻找TSP问题的最优解，即便是在二维欧式空间中，也是NP-hard问题。在实际问题中，一般的求解器会依赖于手工设计的非常精细的启发式算法，来引导搜索过程。即便这些方法在TSP问题中效果很好，情景微微一变，这些方法可能就不在适用，又得重新设计。相比之下，机器学习的方法存在一定的潜能去从数据中挖掘出类似的启发函数，降低人工设计启发算法的开销，也可以不加改变的去解决其他同类组合优化问题。

    目前应用很成功的机器学习方法都是属于有监督的学习。在有监督的学习中，需要训练数据和训练标签，通过梯度下降的方法寻找预测值和标签的损失函数，得到数据到标签的一个映射。但在组合优化问题中，想要得到好的标签本身就很困难，或者某种意义上说，得到了标签也就不需要学习了。因此在组合优化中，更适合一种无监督的学习。

    这篇文章提出一种神经组合优化框架，综合利用循环神经网络和基于策略梯度下降的强化学习方法。一共有两个过程：

    • RL pretraining uses a training set to
      optimize a recurrent neural network (RNN) that parameterizes a stochastic policy over solutions,
      using the expected reward as objective. At test time, the policy is fixed, and one performs inference
      by greedy decoding or sampling

    • active search, involves no pretraining. It starts from a random policy and iteratively optimizes the RNN parameters on a single test instance,
      again using the expected reward objective, while keeping track of the best solution sampled during
      the search.

    中位数和顺序统计量

    在一个由n个元素组成的集合中，第i个顺序统计量是该集合中第i小的元素。这一节讨论从n个互异的元素中选择第i个顺序统计量的问题。一种很直观的想法，先对元素进行排序，然后选出排序后的第i个值，就可以在$\Theta(n\log n)$的时间复杂度内解决问题。其实借助快速排序的思想，可以实现在线性时间内解决这个问题。

    最大值与最小值

    先讨论两种特殊的简单情况。找到一个集合中最大的元素与最小的元素。很简单，我们通过n-1次的比较就可以得出答案，n-1也是上确界。实际上这也是下确界。要找到一个集合中的最大或最小的元素必须要经过n-1次比较。可以参考竞技体育中的一场淘汰赛，有n个选手参加比赛，想要决出冠军必须要进行n-1场比赛。

    如果要同时找到最小值和最大值呢？遍历两遍集合，我们可以在$\Theta(2n-2)$的时间复杂度内解决问题。时间上最多只需要$3n/2$次比较就可以同事确定最小值和最大值。实现方法如下。先初始化一个最小值一个最大值，每次从集合中选取两个元素，先在这两个元素之间进行一次比较，较小者与当前最小值比较，较大者与当前最大值比较，根据比较的结果更新最小值与最大值。这样每两对元素只需要进行３次比较。

    如果n为奇数，则初始的最大值和最小值都为第一个元素的值，后面成对处理。如果初始值为偶数，先将前两个元素进行一次比较，小的作为最小值，大的作为最大值，在成对处理后面的元素。

    第i个顺序统计量的线性时间算法

    排序算法总结

    Posted on 2018-11-08 | Edited on 2019-09-24 | In 数据结构和算法

    排序算法总结

    本文总结常用的７中排序算法，分别是：

    • 插入排序: 插入排序的思想很简单，想象打扑克牌的时候，每次从牌堆中取出一张牌放在左手，每取一张新的牌时和左手中的每一张牌依次比较，放在适当的位置。这样保证左手中的牌始终是有序的。从排序过程容易知道插入排序的最坏时间复杂度和平均时间复杂度都是$\Theta(n^2)$。稳定排序算法

    • 归并排序：归并排序是一种时间复杂度更优的排序算法，可以在$\Theta(nlogn)$的时间复杂度内排序，但需要消耗额外的$\Theta(n)$的空间，不是原址排序算法。归并排序通过递归的将原序列分割成左右两个子序列，再自下向上的合并两个已排好序的子序列。当待排的数据为链表结构时，归并排序可以实现$\Theta(n)$的空间复杂度。稳定排序算法。

    • 堆排序：堆排序是一种最坏情况下时间复杂度为$\Theta(nlogn)$的原址排序算法，使用了堆这种数据结构。堆还可以用来构建优先队列。不稳定排序算法。

    • 快速排序: 快速排序也是一种原址排序算法，期望的运行时间是$\Theta(nlogn)$，但通常在实际情况中比堆排序要快，最坏的情况下是$\Theta(n^2)$。快速排序的代码非常紧凑，运行中隐含的常数系数很小，因此快速排序是排序大数组时的最常用算法。不稳定排序算法。

    上述四种排序算法都是基于比较的排序算法。通过决策树模型可以证明，任何使用比较排序的算法的时间复杂度的下届都是$\Theta(nlogn)$。因此堆排序和归并排序都是渐进最优的比较排序算法。通过比较之外的方法过得输入序列的有效信息就可以突破$\Theta(nlogn)$的下届。下面介绍３种非比较排序的算法：

    • 计数排序: 计数排序假设输入元素的值均在集合${0,1,2,…,k}$中。通过使用数组索引作为确定相对次序的工具。即待排序元素是集合中的第几个。当$k=\Theta(n)$时可以在$\Theta(k+n)$的时间复杂度内将ｎ个元素排好顺序。稳定排序算法。

    • 基数排序: 稳定排序算法。

    • 桶排序: 桶排序假设输入数据出现的概率均匀的分布在[0,1)的半开半闭区间内。

    下面分别分析每种算法的实现与运行时间的分析。

    插入排序

    代码很简单也很紧凑，两层循环，最坏和期望的时间复杂度都是$\Theta(n^2)$。从代码可以看出，插入排序是稳定的排序算法，相同元素的相对顺序不会改变。

    归并排序

    归并排序算法完全遵循分治算法模式，直观上操作如下：

    • 分解: 分解待排序的两个元素的序列为各(n/2)个元素的子序列

    • 解决: 使用归并排序递归的解决两个子序列

    • 合并: 合并两个已排序的子序列以产生需要的答案

    堆排序

    堆(二叉堆)是一个数组，可以看做一个近似完整的二叉树，除了最底层外，该树是完全充满的，并且从左向右填充。在最小堆中，除根节点以外，任意节点的父节点都要小于该节点本身，$A[Parent[i]]<=A[i]$;在最大堆中，任意节点的父节点都要大于等于该节点本身，$A[Parent[i]]>=A[i]$。在堆排序中，我们使用最大堆，最小堆通常用于构造最小优先队列。

    堆结构中涉及的一些基本操作：

    • MAX-Heapify() 维护堆的性质
    • Build-Heapify() 建堆
    • HEAP-INSERT() 优先队列中向堆中插入一个新的元素
    • HEAP-EXTRACT() 优先队列中返回一个堆顶的元素

    在堆排序算法中主要用到了build-heapify() 和　max-heapify()

    上述过程每次向堆中加入一个元素，并维护堆的性质，重要的过程是max-heapify()。在利用一个堆结构保存一个数组数据的时候，堆中叶节点的下表分别是[n/2]+1,…, n。很容易证明。

    为什么下标向下递减而不是从１开始递增？

    将下标值i设置成向下递减是因为，对于节点i，max_heapify()会递归的调整以i为根节点的所有节点，自底向上时，调整上层节点就可以保证之前调整过的节点的大小关系不被破坏，从而可以减小递归次数。

    在max_heapify()执行过程中，每次从A[i],A[2i],A[2i+1]中选出最大的，并将下标存储在largest中。如果largest和i的值不相等，则违背了最大堆的性质，将A[largest]和A[i]互换位置，这样又会影响原以largest为顶点的子堆的性质，因此需要递归的调用程序，使以i为根节点的子堆全都符合最大堆的性质。

    快速排序

    与归并排序一样，快速排序也使用了分治法的思想。下面是对一个典型的子数组进行分治的三步：

    • 分解: 将子数组A[p-r]划分为两个(可能为空)子数组A[p..q-1]和A[q+1..r]，使得第一个数组中的每个元素都<=A[q]同时A[q]<=第二个数组中的每个元素。

    • 解决: 递归调用快速排序，解决子问题A[p..q-1]和A[q+1..r]。

    • 合并: 因为是原址排序的，所以程序结束后A为有序数组。

    对于partition部分，每次循环都维持一个循环不变式：

    • 标号$k\in[p,i]$的部分满足$A[k]<=x$
    • 标号$k\in[i+1,j-1]$的部分满足$A[k]>x$
    • 当k=r时，A[k]=x

    循环开始时，两个集合都为空，循环不变式的前两个条件都满足，第二行的赋值操作满足了第三个条件；循环在中间执行时，有两种情况：当A[j]>x时，只是将j的值＋１，i的值不做变化，这样循环不变式依然可以保持；当A[j]<=x时，将i的值向右移动一位，再把A[j]和Ａ[i]交换位置，使循环不变式成立。循环结束时，将主元交换到i+1的位置，使循环不变式依然成立。显而易见，Partition()的时间复杂度是$\Theta(n)$。

    快速排序的性能取决于划分是否平衡，即与输入数据的分布有关。当划分是平衡的，即每次划分的两个子数组的长度近似相同，那么快速排序的性能可以接近归并排序；如果每次划分的两个子数组不平衡，快速排序的最坏情况则接近于插入排序。

    考虑一种最坏的极端不平衡情况，每次划分时，两个子数组一个长度为０，另一个长度为n-1,这样以来，递归树退化为一条链表，链表的深度为n-1，此时的时间复杂度为$\Theta(n^2)$。

    考虑一种最好的情况，每次划分时两个子数组的长度为[n/2]和[n/2]-1, 此时根据递归式$T(n)=2T(n/2)+\Theta(n)$可以得到时间复杂度为$\Theta(n\log n)$。

    实际上，划分的不平衡性对运行的时间性能的影响并没有看起来的那么显著，只要是常数比例的划分两个子数组，得到的时间复杂度都是$\Theta(n\log n)$。证明过程如下：

    快速排序不是一种稳定的排序算法。

    到目前为止，我们讨论四种比较排序算法，四种算法中除了插入排序，其他三种的平均时间复杂度都可以达到$\Theta(nlgn)$。任何基于比较排序的排序算法，时间复杂度的下届都是$\Theta(nlgn)$，目前已知的所有比较排序算法最多只是在常数因子上优于它。

    计数排序

    计数排序的想法很简单，假设输入的数据都在[1,2,3,…,k]的范围内。对于每个待排序的数x，先计算小于它的个数，然后将它放在后面一个。计数的排序的空间复杂度为$\Theta(k)$, 时间复杂度为$\Theta(k+n)$,当ｋ和n是线性关系时，可以实现线性时间内的排序。

    时间序列预测-ARMA-model

    ARMA模型

    AR(Auto Regression)自回归模型，是一种线性预测模型。用同一变量之前若干时刻的数据来预测现在或未来时刻的数据。这种预测方法只与变量自己有关，而与其他变量无关，所以称作是自回归。

    MA(Moving average)移动平均模型，移动平均模型描述的是自回归部分的误差累计。$\mu$为序列的均值。

    如果某个观察序列可以通过序列预处理判定为平稳非白噪声序列，就可以利用ARMA模型进行建模。利用ARMA模型进行建模的步骤如下：

    • 求出序列的自相关系数(ACF)和偏自相关系数(PACF)。
      1. 若PACF序列满足在p步截尾，且ACF序列被负指数函数控制收敛到0，则Yn为AR(p)序列。
      2. 若ACF序列满足在q步截尾，且PACF序列被负指数函数控制收敛到0，则Yn为MA(q)序列。
      3. 若ACF序列和PACF序列满足皆不截尾，但都被负指数函数控制收敛到0，则Yn为ARMA序列。
    • 根据样本自身相关系数和偏自相关系数选择适当的阶数p,q
    • 如果ACF和PACF都不能收敛，则对原序列做差分，将差分后的序列再做相关分析，知道满足条件。差分以后得到的预测值也为差值，需要还原为原始值。
    • 估计模型中未知参数的值，可以用最小二乘估计，求解Yule-Walker方程，burge法
    • 基于AIC,BIC,或HPIC准则判断模型的好坏。组合了模型预测结果与模型复杂度，防止过拟合。选择出最佳的阶数及其对应的系数。

    在确定模型阶数的时候还有一种时间复杂度比较高的方法，设置一个初始的阶数的上限，可以是待估计序列的长度的线性关系，然后递减遍历每种阶数下对应的AIC,BIC,或HPIC的值，选择最小值对应的阶数。
